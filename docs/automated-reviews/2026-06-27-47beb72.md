## Architecture candidates

### Candidate 1 - Strong

- Files: `extensions/task/task.ts`, `extensions/task/task-prompts.ts`, `tests/task-extension.test.ts`, `docs/handoff/2026-06-27-task-planning-abort-and-cycle-isolation.md`
- Module: task conversation cycle handling inside the `task` module
- Problem: the current module has one broad Interface for menu flow, persisted task state, turn classification, questionnaire cancellation, cycle-boundary filtering, and exit-time marker injection. This diff deepened the Implementation further by adding `stopReason` handling, whole-turn questionnaire scans, and `task-context-end` pairing logic in the same file.
- Solution: carve out a deeper Module around "task conversation cycle" behavior only: classify a turn outcome, close a cycle, and filter completed cycles from context. Keep `/task` command handling and run execution outside that Module.
- Benefits: better Locality for future cancellation bugs, more Leverage from one test surface, and a clearer Seam between task UI flow and conversation-history policy. The deletion test passes: if this logic vanished from `task.ts`, the same complexity would reappear in `agent_end`, `context`, and `/task exit`.

### Candidate 2 - Worth exploring

- Files: `extensions/ui-brand.ts`, `extensions/ui-brand-utils.ts`, `tests/ui-brand-extension.test.ts`, `tests/ui-titlebar.test.ts`
- Module: branded UI runtime state
- Problem: header rendering, footer rendering, title spinner lifecycle, stale-context tolerance, and live usage sampling still live in one Implementation. The Interface is broader than it needs to be because callers must understand title behavior, render behavior, and freshness behavior together.
- Solution: concentrate "current branded UI state" into one deeper Module that materializes cwd/model/usage/status once per render tick, then let header/footer/title adapters consume that snapshot.
- Benefits: improved Locality when session replacement or usage-refresh behavior changes, and more Leverage from smaller render-focused tests instead of end-to-end extension harnesses.
