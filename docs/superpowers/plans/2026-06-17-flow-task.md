# Flow Task Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `/flow` extension V0 that turns explicit flow commands into a strict Task creation, proving, running, and review workflow.

**Architecture:** V0 does not build a fully automatic workflow runtime. `/flow` slash commands send a hidden Flow custom message with `pi.sendMessage(..., { triggerTurn: true })`; the context filter keeps only the current Flow context until `agent_end` and removes stale Flow prompts afterward. Proving and running use the existing `subagent` tool with the `worker` agent; the worker is instructed to read the Task-specific `SKILL.md` and fill the run `todo.md`.

**Tech Stack:** TypeScript ESM, `@earendil-works/pi-coding-agent` extension API, Node built-in `node:test`, existing `subagent` tool and project `agents/worker.md`.

---

## Scope

Implement the first useful slice of Flow:

- Parse `/flow` commands.
- Send strict hidden Flow workflow prompts for create, prove, run, review, and status.
- Add reusable prompt builders that tell the main agent exactly which files to create or update.
- Register the extension from `extensions/index.ts`.
- Add tests for parsing, prompt generation, command registration, hidden Flow message dispatch, context filtering, and `/ugk` command listing.

Do not implement DAG, parallel fan-out/fan-in, automatic validator execution, or a new subagent process runner in this plan. V0 uses the current main-agent-plus-subagent-tool loop.

```ts
import type { FlowRequest } from "./types.ts";

function unquote(value: string): string {
	const trimmed = value.trim();
	if ((trimmed.startsWith('"') && trimmed.endsWith('"')) || (trimmed.startsWith("'") && trimmed.endsWith("'"))) {
		return trimmed.slice(1, -1).trim();
	}
	return trimmed;
}

function splitInlineInput(rest: string): { taskId: string; input?: string } | null {
	const match = rest.trim().match(/^(\S+)(?:\s+--input\s+([\s\S]+))?$/);
	if (!match) return null;
	const input = match[2] ? unquote(match[2]) : undefined;
	return { taskId: match[1], input };
}

export function parseFlowCommand(args: string): FlowRequest {
	const text = args.trim();
	if (!text) return { kind: "help" };

	if (text === "status") return { kind: "status" };

	const createPrefix = "task create";
	if (text === createPrefix) {
		return { kind: "error", message: 'Usage: /flow task create "自然语言目标"' };
	}
	if (text.startsWith(`${createPrefix} `)) {
		const goal = unquote(text.slice(createPrefix.length));
		if (!goal) return { kind: "error", message: 'Usage: /flow task create "自然语言目标"' };
		return { kind: "task-create", goal };
	}

	const provePrefix = "task prove";
	if (text === provePrefix) {
		return { kind: "error", message: "Usage: /flow task prove <task-id> [--input <inline-input>]" };
	}
	if (text.startsWith(`${provePrefix} `)) {
		const parsed = splitInlineInput(text.slice(provePrefix.length));
		if (!parsed) return { kind: "error", message: "Usage: /flow task prove <task-id> [--input <inline-input>]" };
		return { kind: "task-prove", ...parsed };
	}

	const runPrefix = "run";
	if (text === runPrefix) return { kind: "error", message: "Usage: /flow run <task-id> [--input <inline-input>]" };
	if (text.startsWith(`${runPrefix} `)) {
		const parsed = splitInlineInput(text.slice(runPrefix.length));
		if (!parsed) return { kind: "error", message: "Usage: /flow run <task-id> [--input <inline-input>]" };
		return { kind: "task-run", ...parsed };
	}

	const reviewPrefix = "task review";
	if (text === reviewPrefix) return { kind: "error", message: "Usage: /flow task review <run-id>" };
	if (text.startsWith(`${reviewPrefix} `)) {
		const runId = text.slice(reviewPrefix.length).trim();
		if (!runId) return { kind: "error", message: "Usage: /flow task review <run-id>" };
		return { kind: "task-review", runId };
	}

	return { kind: "help" };
}
```

```ts
import type { FlowRequest } from "./types.ts";

export function buildFlowHelpText(): string {
	return [
		"Flow commands:",
		'  /flow task create "目标"',
		"  /flow task prove <task-id> [--input <inline-input>]",
		"  /flow run <task-id> [--input <inline-input>]",
		"  /flow task review <run-id>",
		"  /flow status",
	].join("\n");
}

function createPrompt(goal: string): string {
	return `[FLOW TASK CREATE]
你正在创建一个 Flow Task。Flow 是显式严格模式，普通聊天不进入这套流程。

用户自然语言目标：
${goal}

你的职责：
1. 根据目标推断一个简短稳定的 task id，例如 x-search-post-collector。
2. 创建以下文件：
   - .flow/tasks/<task-id>/task.json
   - .flow/tasks/<task-id>/SKILL.md
   - .flow/tasks/<task-id>/todo.template.md
   - .flow/tasks/<task-id>/input.schema.json
   - .flow/tasks/<task-id>/output.schema.json
   - .flow/tasks/<task-id>/validator.md
   - .flow/tasks/<task-id>/runs/
3. task.json 的 status 必须是 draft，version 必须是 1。
4. SKILL.md 必须采用“最优路径 A/B/C/D + 每步动作/注意事项/证据/失败处理”结构。
5. todo.template.md 必须与最优路径一一对应，并包含原计划、实际执行、偏离旧方案、解决过程、证据、复盘候选。
6. validator.md 可以先写验证说明或人工门禁说明。
7. 不要把 Task 标记为 active。create 只生成草案，不代表可复用。

完成后输出：
- task id
- 创建的文件
- 哪些假设需要用户确认
- 下一步命令：/flow task prove <task-id> --input "真实样例输入"`;
}

function provePrompt(taskId: string, input?: string): string {
	const inputLine = input ? `本次样例输入：\n${input}` : "本次样例输入：用户尚未提供。如果缺少必要输入，先问一个关键问题。";
	return `[FLOW TASK PROVE]
你正在证明一个 draft/proving Task 是否能跑通。

Task id：${taskId}
${inputLine}

你的职责：
1. 读取 .flow/tasks/${taskId}/task.json，确认 Task 存在。
2. 如果 status 是 draft，把它更新为 proving。
3. 创建新的 .flow/tasks/${taskId}/runs/run-<next-number>/ 目录。
4. 写入 input.json，记录本次真实样例输入。
5. 从 todo.template.md 复制本次 todo.md。
6. 使用 subagent 工具启动隔离 driver，推荐 agent="worker"。
7. driver 任务必须要求：
   - 读取当前 Task 的 \`SKILL.md\`
   - 读取本次 \`input.json\`
   - 按 \`todo.md\` 逐项执行并填写
   - 保存 output/ 和 evidence/
   - 不做最终复盘
8. driver 返回后，main agent 读取 todo.md、output/、evidence/。
9. 按 validator.md 做验证记录，写入 validation.md 和 status.json。
10. 如果通过，把 run 标记 passed，并把 Task 标记 verified。
11. 如果失败且可以修复，进入 resolving 并尝试一次修复；修复后通过则标记 fixed。
12. 如果无法修复，标记 needs-human，保留证据，不更新 SKILL.md。

禁止：
- 不允许 driver subagent 自己决定更新 SKILL.md。
- 不允许失败未解决时写回 task skill。
- 不允许只口头说通过，必须有 todo、output、evidence、validation。`;
}

function runPrompt(taskId: string, input?: string): string {
	const inputLine = input ? `本次输入：\n${input}` : "本次输入：用户尚未提供。如果缺少必要输入，先问一个关键问题。";
	return `[FLOW TASK RUN]
你正在正式运行一个已 verified/active 的 Flow Task。

Task id：${taskId}
${inputLine}

执行规则：
1. 读取 .flow/tasks/${taskId}/task.json。
2. 如果 status 是 draft，停止并提示先运行 /flow task prove ${taskId}。
3. 如果 status 是 needs-human，停止并提示先 review 或补充用户指导。
4. 创建新的 run 目录，复制 todo.template.md 为 todo.md。
5. 用 subagent 工具启动 worker driver。
6. driver 必须读取 Task 的 SKILL.md，按 todo.md 执行并保存 output/evidence。
7. main agent 负责验证、记录 status，不让 driver 自评复盘。
8. passed 或 fixed 后可以提示用户运行 /flow task review <run-id>。`;
}

function reviewPrompt(runId: string): string {
	return `[FLOW TASK REVIEW]
你正在主持 Flow run 复盘。复盘不能由 driver subagent 自评。

Run id：${runId}

你的职责：
1. 定位包含该 run id 的 .flow/tasks/<task-id>/runs/${runId}/。
2. 读取同级 Task 的 SKILL.md、todo.template.md、task.json。
3. 读取本次 run 的 todo.md、output/、evidence/、validation.md、feedback.md、status.json。
4. 按 SKILL.md 的最优路径 A/B/C/D 逐环节向用户核对：
   - 这一步是否遇到问题？
   - 是否偏离旧方案？
   - 最后怎么修正？
   - 这个修正是否应该沉淀？
   - 应该更新 SKILL.md、todo.template.md、validator.md，还是只归档？
5. 写入 review.md，列出每个步骤的结论和建议修改。
6. 只有用户确认，且 run 状态是 passed 或 fixed，才能修改 SKILL.md、todo.template.md 或 validator.md。
7. 写回后 bump task.json.version 并更新 updated_at。

禁止：
- 不要把 failed 或 needs-human 的未解决经验写回 skill。
- 不要把偶发现象写成强规则。
- 不要让 driver subagent 自己输出最终复盘结论。`;
}

function statusPrompt(): string {
	return `[FLOW STATUS]
读取 .flow/tasks/ 下的 Task 状态，输出简洁表格：
- task id
- status
- version
- 最近 run
- 下一步建议

如果 .flow/tasks/ 不存在，说明当前项目还没有 Flow Task，并提示：
/flow task create "目标"`;
}

export function buildFlowRequestPrompt(request: FlowRequest): string {
	switch (request.kind) {
		case "task-create":
			return createPrompt(request.goal);
		case "task-prove":
			return provePrompt(request.taskId, request.input);
		case "task-run":
			return runPrompt(request.taskId, request.input);
		case "task-review":
			return reviewPrompt(request.runId);
		case "status":
			return statusPrompt();
		case "error":
			return request.message;
		case "help":
			return buildFlowHelpText();
	}
}
```

```ts
	let activeContextId: string | undefined;

	pi.registerCommand("flow", {
		description: "Flow Task workflow: create, prove, run, review reusable task skills",
		handler: async (args, ctx) => {
			const request = parseFlowCommand(String(args ?? ""));
			if (request.kind === "help") {
				ctx.ui.notify(buildFlowHelpText(), "info");
				return;
			}
			if (request.kind === "error") {
				ctx.ui.notify(request.message, "error");
				return;
			}
			if (!ctx.isIdle()) {
				ctx.ui.notify("Flow request requires the agent to be idle.", "warning");
				return;
			}

			const contextId = `flow-${++nextContextId}`;
			activeContextId = contextId;
			pi.sendMessage(
				{
					customType: "flow-task-context",
					content: `${buildFlowRequestPrompt(request)}\n\n[FLOW CONTEXT ID: ${contextId}]`,
					display: false,
				},
				{ triggerTurn: true },
			);
			ctx.ui.notify(formatFlowQueued(request), "info");
		},
	});

	pi.on("context", async (event) => ({
		messages: event.messages.filter((message) => {
			const msg = message as AgentMessage & { customType?: string };
			if (msg.customType === "flow-task-context") {
				return activeContextId !== undefined && String(msg.content).includes(`[FLOW CONTEXT ID: ${activeContextId}]`);
			}
			return !hasFlowPromptMarker(msg);
		}),
	}));

	pi.on("agent_end", async () => {
		activeContextId = undefined;
	});
}
```
